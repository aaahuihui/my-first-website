# 云端文件存储方案

这份文档用于记录当前网站配套使用的云端资料管理方案。

当前主用方案：
- 主云盘：阿里云盘
- 备用方案：百度网盘
- 网站仓库只保存代码、说明和下载入口
- 大文件长期保存到云盘中，不直接塞进网站仓库

## 推荐的云盘根目录结构

建议在阿里云盘里建立一个总文件夹：

`Li_Yinhui_Cloud_Library`

然后在这个总文件夹下按下面结构分类：

1. `00_Inbox_待整理`
- 新上传但还没分类的文件先放这里
- 每周或每次上传后再整理到正式目录

2. `01_课程资料`
- 存放课程 PDF、讲义、作业说明、复习资料
- 推荐子目录：
  - `课程名\课件`
  - `课程名\作业`
  - `课程名\笔记`

3. `02_汇报材料`
- 存放 PPT、答辩材料、展示文档
- 推荐子目录：
  - `课程汇报`
  - `项目汇报`
  - `研究汇报`

4. `03_项目文件`
- 存放项目压缩包、代码说明、演示文档
- 推荐子目录：
  - `项目名\源码压缩包`
  - `项目名\发布文件`
  - `项目名\说明文档`
  - `项目名\展示材料`

5. `04_常用文档`
- 存放简历、报名表、模板、常用说明文档
- 推荐子目录：
  - `简历`
  - `模板`
  - `常用表格`

6. `05_研究方向`
- 存放人工智能、机器学习、计算机视觉相关资料
- 推荐子目录：
  - `人工智能_机器学习`
  - `计算机视觉`
  - `阅读笔记`
  - `论文资料`

7. `06_个人归档`
- 按年份归档旧文件，避免主页下载区越来越乱
- 推荐子目录：
  - `2026`
  - `2027`

8. `90_临时分享`
- 用于临时外发的资料
- 不建议长期堆放
- 定期清理

9. `99_重要备份索引`
- 用于记录最重要资料是否还有第二份备份
- 可以放一个简短文档，记录哪些文件还同步保存在本地硬盘或其他云盘

## 推荐的实际使用规则

1. 新文件先放 `00_Inbox_待整理`
2. 整理完成后移动到正式分类目录
3. 对外分享的文件尽量从正式目录复制到 `90_临时分享`
4. 特别重要的文件保留第二份备份
5. 网站中的下载页只挂链接，不直接保存大文件

## 网站接入资料时建议提供的信息

以后如果要把资料接到网站的 `downloads.html`，建议每条资料至少准备这些信息：

- 分类：课程资料 / 汇报材料 / 项目文件 / 常用文档 / 研究资料
- 文件名
- 简短说明
- 阿里云盘分享链接
- 百度网盘分享链接（可选）
- 提取码（如果有）
- 更新时间

## 当前结论

- 先用阿里云盘作为主存储是合适的
- 百度网盘作为备用分享渠道也很实用
- 网站负责展示入口，云盘负责长期保存
