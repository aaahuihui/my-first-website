现在最推荐的维护方式已经变成了：先启动本地后端，再打开本地管理页录入资料，而不是直接手改网页。

适合你自己维护资料列表。运行 npm run dev 后，访问 http://127.0.0.1:3000/manage.html 即可新增、查看、删除资料。

推荐的云盘分类

00_Inbox_待整理

新上传但还没分类的文件先放这里，后续再整理到正式目录。

01_课程资料

存放课程 PDF、讲义、作业说明、复习资料，建议按课程名继续细分。

02_汇报材料

存放课程汇报、项目汇报、研究汇报 PPT 和展示材料。

03_项目文件

存放项目压缩包、发布文件、说明文档和演示材料。

04_常用文档

存放简历、模板、表格和常用文档，方便后续快速调用。

05_研究方向

存放人工智能、机器学习、计算机视觉相关资料、笔记和论文。

06_个人归档

按年份归档旧资料，避免主目录越来越乱。

90_临时分享 / 99_重要备份索引

一个用于对外临时分享，一个用于记录最重要资料的双备份状态。

后端模式下的维护思路

  • 网页仓库继续保存前端页面和说明
  • SQLite 数据库保存资料标题、说明、分类、链接和更新时间
  • 管理页负责新增、查看和删除资料
  • 公开资料显示在 `downloads.html`，私人资料显示在 `private.html`
  • 真正的大文件继续长期放在阿里云盘或百度网盘中
